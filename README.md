# ابراهيم ماركت

واجهة Frontend فقط لتطبيق كاشير بسيط لمحل صغير. التطبيق مبني بـ Vite + React + TypeScript + Tailwind CSS، ويدعم العربية بالكامل مع اتجاه RTL.

## التشغيل

```bash
npm install
npm run dev
```

بعد التشغيل افتح الرابط الذي يظهر في الطرفية، غالبًا:

```text
http://localhost:5173
```

للتأكد من جاهزية البناء:

```bash
npm run build
```

## طريقة عمل الباركود

من صفحة `/cashier` استخدم حقل `امسح الباركود هنا` في أعلى الصفحة. اكتب أو امسح باركود منتج موجود ثم اضغط Enter.

أمثلة جاهزة:

```text
622100001001
622100001002
622100001003
```

إذا كان المنتج موجودًا يتم إضافته إلى الفاتورة الحالية. إذا كان موجودًا بالفعل داخل الفاتورة تزيد الكمية. إذا لم يكن الباركود موجودًا تظهر رسالة خطأ عربية.

## أماكن بيانات Mock

كل البيانات التجريبية موجودة داخل:

```text
src/data/mockProducts.ts
src/data/mockCustomers.ts
src/data/mockInvoices.ts
```

كل العمليات الحالية تتم داخل React state فقط، بدون Backend، بدون Database، وبدون `fetch`.

## الربط لاحقًا مع Backend API

إدارة البيانات مجمعة في:

```text
src/store/AppStore.tsx
```

عند إضافة Backend لاحقًا يمكن استبدال منطق القراءة والكتابة داخل هذا الملف بطبقة API أو service functions، مع الحفاظ على الصفحات والمكونات كما هي قدر الإمكان. الأنواع الأساسية موجودة في:

```text
src/types/index.ts
```

والحسابات المشتركة موجودة في:

```text
src/utils/calculations.ts
src/utils/formatCurrency.ts
```
